# 📅 April 2026

## ✨ Improvements

- **Configurable Filename for Report Downloads in Lists**  
  You can now configure a custom filename for report downloads in lists using the `web:DownloadFilename` property on the list tag. Static names and expressions are supported — for example, you can combine filter parameters and entity values to build meaningful filenames automatically. If the property is omitted, the previous default behavior (list name + date) still applies.

- **Display Used Profile in List**  
  Lists now show the active formatting profile name instead of the generic "Profile" label. This makes it easier to see at a glance which profile is applied to the current list view.

- **"Markdownize" Formatting Profile Info**  
  The `APP_CaptionExpression` field in formatting profiles now renders Markdown. Captions and descriptions can use headings, bold text, lists, and other Markdown elements to improve readability.

- **Seamless Time Entry Docking**  
  When creating a new time entry with the **+** button or **Ctrl+I**, the entry now starts exactly at the end of the previous entry without rounding to the nearest 15 minutes. This makes it easier to log time continuously without gaps or overlaps.

- **Signal Improvements**  
  Two usability improvements for the Signals panel:
  - Double-clicking a signal now opens a new time entry pre-filled with the signal's duration.
  - The signal time summary now shows the actual worked duration instead of a min/max range.

- **Updated Dialog When Deleting a Time Cockpit User**  
  The dialog shown when attempting to delete a user has been revised. It now explains that direct deletion is not supported and guides administrators to use the **Active** checkbox to deactivate a user instead.

- **Right-Click on Group Logo in Lists**  
  Right-clicking on a group logo in a list now opens the context menu, consistent with right-clicking the group row itself.

## 🐛 Bug Fixes

- **URL Encoding in Conditions**  
  URL-encoded characters in list conditions are now handled correctly.

- **Excel Export: Report Name Headline Spans All Columns**  
  When exporting a list to Excel via the print view, the report name headline now spans across all data columns. Previously it was clipped to the configured page width. The headline is also left-aligned, consistent with other header rows.

- **"New Version Available" Notification Shown Less Frequently**  
  The "New version available" notification was appearing too often. It is now shown less frequently to avoid unnecessary interruptions during work.

- **Closing TimesheetAdminForm Clears User Selection**  
  Fixed an error that occurred when removing the user selection in the relation cell of the TimesheetAdminForm. The `HasPermission` function now handles a null entity object gracefully, preventing the error.
