### What should I do on my first day using time cockpit?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

**Your First Time Cockpit Session:**

**1. Log In**
- Use credentials provided by HR
- Confirm you can access the system

**2. Navigate to Time Sheet Calendar**
- Click **Timesheet** in the main menu
- Opens the graphical calendar view
- This is where you'll track your time

**3. Create Your First Entry**
- Click in the calendar to create a new entry
- Or click **New Timesheet Entry** button
- Fill in:
  - **Begin Time:** When you started work (e.g., 9:00 AM)
  - **End Time:** Or enter **Duration** instead
  - **Description:** What you worked on
  - **Customer/Project/Task:** If applicable (ask your manager)
- Click **Save**

**4. Understand Your Schedule**
- Ask your manager:
  - What projects should I track time to?
  - How detailed should my descriptions be?
  - Should I track breaks?
  - When are timesheets due?

**5. Set Up Your Routine**
- Decide when to log time:
  - **Real-time:** Track as you work (start/stop entries)
  - **End of day:** Fill in at day's end (while fresh in memory)
  - **Weekly:** Once per week (less accurate, but some prefer it)
- Set reminder (phone/calendar) to track time

**First Week Checklist:**
```
□ Successfully logged in
□ Created first timesheet entry
□ Understand what projects to assign time to
□ Know expected hours per day/week
□ Set up reminder to track time daily
□ Know who to ask questions (manager, HR, or colleague)
```

**Common First-Week Questions:**

"What if I don't know which project to select?"
→ Ask your manager or select "Internal" for training time

"Do I log lunch breaks?"
→ Typically no, but ask your company's policy

"How often should I save entries?"
→ Save after creating each entry or at end of day

## Daily Time Tracking

### How do I quickly track my time each day?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

**Quick Daily Workflow Options:**

**Method 1: Morning Planning (5-10 minutes)**
1. Open time cockpit at start of day
2. Review your calendar/task list
3. Create timesheet entries for planned work:
   - Morning project work: 9:00-12:00
   - Afternoon meetings: 1:00-3:00
   - Email/admin time: 3:00-5:00
4. Adjust throughout the day as plans change

**Method 2: End-of-Day Logging (10-15 minutes)**
1. At end of workday, open time cockpit
2. Think back through your day
3. Create entries for major blocks of time:
   - Project A work: 9:00-11:30
   - Team meeting: 11:30-12:00
   - Project B work: 1:00-4:00
   - Administrative: 4:00-5:00
4. Save all entries

**Method 3: Real-Time Tracking (As You Work)**
1. Start task → Immediately create timesheet entry
2. Set begin time, leave end time open (if supported)
3. Finish task → Set end time
4. Start next task → Create new entry
5. Most accurate but requires discipline

**Quick Entry Tips:**

**Use Previous Entries:**
- Find yesterday's entry for same work
- Click **Copy** or Ctrl+Drag to duplicate
- Change date and times
- Saves reselecting project/task

**Use Templates:**
- Create timesheet template for recurring work:
  - "Daily Standup" (30 min daily)
  - "Email & Communication" (1 hour daily)
- Apply template instead of creating from scratch

**Keyboard Shortcuts (If Available):**
- Ctrl+N: New entry
- Ctrl+S: Save
- Arrow keys: Navigate dates
- *Check documentation for your version's shortcuts*

**Mobile Quick Entry:**
- Open time cockpit on phone
- Tap **+** to create entry
- Enter basics quickly (full details later at desk)
- Saves time when on the go

**Weekly Batch Entry (Not Recommended But Practical):**

If you can't track daily:
1. Set aside 30 min Friday afternoon
2. Review your calendar/notes for the week
3. Reconstruct your week's activities
4. Enter all timesheets for Mon-Fri
5. **Downside:** Less accurate, easy to forget details

**What to Track:**

**Do Track:**
- ✓ Project/client work (largest use of time)
- ✓ Meetings (team, client, 1-on-1s)
- ✓ Email/communication (if substantial)
- ✓ Administrative tasks
- ✓ Training/learning

**Don't Track (Usually):**
- ✗ Lunch break (unpaid time)
- ✗ Coffee breaks (unless long)
- ✗ Personal time
- *Check your company's policy*

**Target Detail Level:**

**Good Description:**
- "Developed login feature for Customer Portal"
- "Client meeting - Q1 planning"
- "Code review for Project X"

**Too Vague:**
- "Work"
- "Stuff"
- "Meeting"

**Too Detailed:**
- "Wrote code on lines 145-233 of auth.js implementing password validation using bcrypt..."

**Aim for:** Someone reading your timesheet understands what you did.

### What if I forget to log time for a day?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

Don't worry - you can always add time retroactively:

**Adding Past Entries:**

**1. Navigate to the Date**
- Open Time Sheet Calendar
- Use date navigation (arrows or date picker)
- Go back to the missing day

**2. Create Entries**
- Click in the calendar or click **New**
- Set the past date
- Enter begin/end times or duration
- Add description and projects
- Save

**No Deadline Yet?**
- You can add/edit entries any time before they're needed for billing or payroll
- Many companies have weekly or monthly deadlines

**Tips for Remembering:**

**Use Your Calendar:**
- Review Outlook/Google Calendar appointments
- Meetings = timesheet entries
- Scheduled blocks = what you worked on

**Use Email:**
- Sent emails show what projects you worked on
- Timestamps indicate when you were working

**Use Notes:**
- Daily planner
- Project management tools (Jira, Asana)
- Code commits (for developers)
- Documents saved/edited

**Set Reminders:**
- Phone alarm at 4:30 PM: "Log your time"
- Calendar event: "Update timesheet" (daily or weekly)
- Habit stacking: "Before leaving office, log time"

**Make It a Routine:**

**Daily habit** (recommended):
- Every day at same time (EOD works well)
- Just 5-10 minutes
- Much easier than weekly reconstruction

**Weekly habit:**
- Friday afternoon: Review and fill gaps
- Takes longer (30-60min) but still manageable

**What If It's Been Weeks?**

**Long Gap Reconstruction:**
1. Don't panic - it's recoverable
2. Set aside focused time (1-2 hours)
3. Use all available evidence:
   - Email sent/received
   - Calendar appointments
   - Project notes/commits
   - Colleagues' memory
4. Create best estimate entries
5. Note if uncertain: "EST: Project A work"
6. Learn from it - set better reminders

**Company Policies:**

**Cutoff Dates:**
- Many organizations have deadlines:
  - "All previous week time due by Monday 10 AM"
  - "Month end by 5th of next month"
- Late entries may delay:
  - Your payroll
  - Client billing
  - Manager's reporting
- Aim to stay current

### How do I track time when I work on multiple things in a day?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

Most work days involve switching between tasks - here's how to track accurately:

**Multiple Entries Approach:**

**Create separate timesheet entries for each significant activity:**

**Example Day:**
```
9:00 - 10:30    Project A - Development work (1.5h)
10:30 - 11:00   Team Standup Meeting (0.5h)
11:00 - 12:30   Project B - Client Review (1.5h)
(12:30 - 1:30   Lunch - not tracked)
1:30 - 3:00     Project A - Development cont'd (1.5h)
3:00 - 4:00     Email & Admin (1h)
4:00 - 5:00     Project C - Planning (1h)

Total: 7.5 hours across 6 entries
```

**Each task gets its own entry** with distinct:
- Begin/end times
- Project assignment
- Description

**Handling Frequent Switching:**

**Short Interruptions (< 15 min):**
- **Don't create new entries** for brief interruptions
- Example: 5-minute question from colleague during Project A work
- **Keep in existing entry:** Still tracking Project A

**Medium Tasks (15 min - 1 hour):**
- **Judgment call:** Depends on importance
- Significant meeting: Separate entry
- Quick email check: Combine with adjacent work

**Small Tasks (5-15 min):**
- **Option 1: Round into adjacent blocks:**
  - Working on Project A, 10-min email, back to Project A
  - Just track longer Project A block
- **Option 2: Create "Admin" bucket:**
  - Collect small tasks into single "Admin/Email" entry

**Concurrent Work:**

**If truly multitasking** (rare):
- Example: Attending meeting while monitoring production system
- **Pick primary activity:** The meeting
- Note in description: "During production monitoring"

**Grouping Strategies:**

**By Project:**
All time on Project A lumped together:
```
9:00 - 12:00   Project A (3h total, but actually had interruptions)
```
**Pro:** Simpler
**Con:** Less accurate

**By Time Block:**
Exact start/end for every distinct activity:
```
9:00 - 9:45    Project A - Feature development
9:45 - 10:00   Email
10:00 - 10:30  Project A - Code review
10:30 - 11:00  Team meeting
...
```
**Pro:** Most accurate
**Con:** More time-consuming to log

**By Activity Type:**
```
Engineering Work - Project A: 4h
Meetings: 2h
Communication/Admin: 1.5h
```
**Pro:** Captures effort distribution
**Con:** Loses project granularity

**Choose based on your company's needs** - ask your manager.

**Overlapping Entries:**

**Avoid overlaps:**
- Time cockpit visualizes overlaps (stacked entries)
- Indicates error (can't work on two things simultaneously)
- **Fix:** Adjust times so entries don't overlap

## Vacation & Time Off

### How do I request vacation?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

**Step-by-Step Vacation Request:**

**1. Check Your Balance**
- Navigate to **User** → **User Details**
- Find your record (may bepre-filtered to you)
- Look for vacation balance fields:
  - **Yearly Entitlement:** Total days per year
  - **Used:** Days taken so far
  - **Remaining:** Available balance
- Ensure you have enough days

**2. Create Vacation Request**
- Navigate to **User** → **Vacation**
- Click **New**
- Fill in the form:
  - **User Detail:** Your name (should be pre-filled)
  - **Begin Date:** First day off
  - **End Date:** Last day off (inclusive)
  - **Vacation Type:** Full day, half-day AM, half-day PM
  - **Description:** Optional (e.g., "Family trip to beach")
- Click **Save**

**3. Request is Submitted**
- Your manager/department lead receives notification (if configured)
- Request shows as **Pending** (Is Approved = False)
- Wait for approval

**4. Check Approval Status**
- Return to **User** → **Vacation**
- Find your request
- Check **Is Approved** column:
  - Empty/Unchecked = Still pending
  - Checkmark = Approved ✓
  - May have **Approved Timestamp** and **Approver** name

**5. Approved Vacation Appears in Calendar**
- Open **Time Sheetcalendar**
- Approved vacation shows as blocked time
- You cannot create timesheet entries during vacation days

**When to Submit:**

**Advance Notice:**
- Most companies require 1-2 weeks notice minimum
- Longer vacations (> 1 week): 3-4 weeks notice
- Holiday periods: Submit early (months ahead)
- Check your company's policy

**Last-Minute/Emergency:**
- Contact manager directly first (phone/email)
- Then submit formal request in system
- Note reason in description field

**Tips for Approval:**

**Increase Approval Likelihood:**
- ✓ Submit well in advance
- ✓ Check team calendar (avoid conflicts with colleagues)
- ✓ Avoid known busy periods (project deadlines, year-end)
- ✓ Communicate with team about coverage
- ✓ Have sufficient vacation balance

**Communication:**

**Notify Your Team:**
- Even after approval, remind team:
  - Email: "FYI - I'll be out Aug 15-19"
  - Calendar: Block Outlook/Google Calendar (if separate)
  - Slack/Teams: Set status to vacation
  - Handoff: Brief colleagues on urgent matters

**What If Rejected?**

**If denied:**
1. Your manager should communicate reason
2. Discuss alternative dates
3. Address concerns (coverage, timing)
4. Resubmit adjusted request

### Can I cancel or change my vacation request?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

Yes, you can modify or cancel vacation requests:

**Before Approval:**

**Editing:**
1. Navigate to **User** → **Vacation**
2. Find your pending request
3. Open it (double-click or select)
4. Change dates or details
5. Save

**Canceling:**
1. Find your pending request
2. Select it
3. Click **Delete** or trash icon
4. Request is removed

**After Approval (But Before Taking Vacation):**

**If plans change:**
1. **Talk to your manager first** - explain situation
2. Options depend on company policy:

**Option A: Delete Approved Request**
- If allowed, simply delete the vacation record
- Your vacation balance is restored
- Calendar blockage removed

**Option B: Manager Un-Approves**
- Manager opens your vacation record
- Unchecks **Is Approved**
- You can then edit or delete

**Option C: Create New Request**
- Delete old request
- Create new request with correct dates
- Resubmit for approval

**During or After Vacation:**

**If you already took the time:**
- Cannot cancel (you were actually absent)
- If you worked during approved vacation (emergency):
  - Contact manager/HR
  - They may adjust the vacation record
  - You may need to create timesheet entries for work performed

**Reducing Vacation Length:**

**Scenario:** Requested 2 weeks, only taking 1 week

1. Edit the vacation record
2. Adjust **End Date** to earlier date
3. Save
4. Unused days remain in your balance (if policy allows)

**Extending Vacation:**

**Scenario:** Want to extend your vacation

1. Can't extend existing approved request
2. Create **new request** for additional days
3. Submit for approval
4. Manager sees context (extension of existing vacation)

**Communication:**

**Always notify:**
- Your manager (first priority)
- Team members (affected parties)
- Update calendar/status
- Document any changes

### What do I do if I'm sick?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

Sick leave procedure differs from vacation (unplanned vs. planned):

**When You're Too Sick to Work:**

**1. Notify Your Team (Urgent - Same Day)**
- **Call or message your manager** ASAP (before work start time)
  - Phone call preferred (don't just email)
  - Or text/message if that's your team's norm
- Simple message: "I'm sick today and won't be able to work."
- Mention if you might be out multiple days: "I'll update you tomorrow on status."

**2. Log Sick Leave in Time Cockpit (When Able)**
- May be same day or when you're feeling better
- Navigate to **User** → **Sick Leave**
- Click **New**
- Fill in:
  - **User Detail:** Your name
  - **Begin Date:** First sick day
  - **End Date:** Last sick day (or leave open if unsure)
  - **Description:** Optional (respect your privacy - "Illness" is sufficient)
- Click **Save**

**3. Manager Approves Retroactively**
- Your manager or HR sees the entry
- Approves it for record-keeping
- Appears in calendar as sick leave (different from vacation)

**4. When You Return**
- Update sick leave **End Date** if needed
- Resume normal time tracking

**Key Differences from Vacation:**

| Vacation | Sick Leave |
|----------|------------|
| Planned | Unplanned |
| Request in advance | Notify same-day |
| Approved before | Approved after |
| Consumes vacation balance | Separate (doesn't use vacation) |

**Extended Illness (Multiple Days):**

**Day 1:**
- Call in sick (as above)
- Create sick leave entry (single day or estimated range)

**Each Following Day:**
- Check in with manager (brief update)
- Update **End Date** when you know return date

**Longer Than 3-5 Days:**
- May require doctor's note (check company policy)
- Coordinate with HR
- May involve short-term disability process

**Sick Child / Family Member:**

**Company policy dependent:**
- Some allow sick leave for caring for family
- Others require using vacation
- Check your employee handbook or ask HR

**Part-Day Sickness:**

**Scenario:** Feel sick midday, leave early

**Option 1:** Sick leave for full day
- Create sick leave entry for the day
- Don't create timesheet entries

**Option 2:** Split the day
- Timesheet entries for morning hours worked
- Sick leave for afternoon (half-day)
- Ask your manager which approach to use

**Doctor Appointments:**

**Planned medical appointments:**
- May use sick leave or vacation (policy dependent)
- Some companies: Sick leave for appointments
- Submit request in advance if possible

**Privacy:**

**You don't need to share medical details:**
- "Illness" or "Medical" is sufficient description
- HIPAA/privacy laws protect your health information
- Manager may ask general questions (contagious? when return?) but not diagnosis

### What if I have technical problems or can't log in?

> [!AI-GENERATED]
> This content is AI-generated and requires verification.

**Common Issues & Solutions:**

**Can't Log In:**

**"Invalid username or password"**
- Double-check username (usually your email)
- Check caps lock is off
- For Microsoft accounts: Reset at account.microsoft.com
- For time cockpit accounts: Contact your HR/IT administrator

**"Account not found"**
- Verify you're using correct email address
- Check with HR that account is set up
- New employees: Account may not be created yet

**Multi-Factor Authentication (MFA) issues:**
- Azure AD accounts may require MFA
- Ensure you have authenticator app set up
- Contact IT if MFA not working

**Page Won't Load:**

**Blank screen or loading forever:**
- Refresh the page (F5 or Ctrl+R)
- Clear browser cache:
  - Chrome: Settings → Privacy → Clear browsing data
  - Firefox: Options → Privacy → Clear Data
- Try different browser
- Check internet connection

**"Can't connect to server"**
- Check your internet connection
- Try accessing other websites (is your WiFi working?)
- Time cockpit service may be down (rare) - check with colleagues
- Firewall/VPN issues - contact IT

**Browser Compatibility:**

**Recommended Browsers:**
- Chrome (recommended)
- Firefox
- Microsoft Edge
- Safari (Mac/iOS)

**Update browser to latest version** if having issues

**Performance Issues:**

**Slow loading:**
- Close other browser tabs
- Disable browser extensions temporarily
- Check computer performance (high CPU/memory usage?)
- Large lists load slowly - apply filters

**Can't Save Entries:**

**"Error saving" or nothing happens:**
- Check required fields are filled in
- Check internet connection (saves to cloud)
- Try refreshing page and re-entering
- Contact support if persistent

**Data Not Appearing:**

**"Where are my timesheet entries?"**
- Check date range (are you viewing correct week?)
- Check user selection (are you viewing someone else's calendar?)
- Check filters (did you accidentally filter entries out?)
- Try clearing all filters

**1. Check Documentation**

**2. Ask a Colleague**
- Someone who uses time cockpit successfully

- Department admin

**3. Contact Internal Support**
- Your company's time cockpit administrator
- HR department
- IT helpdesk

**4. Contact Time Cockpit Support**

  - Your name and company

  - Browser and operating system

**Emergency: Can't Track Time by Deadline:**
